# SESSION-IMPLEMENTAR-US - Execucao de User Story em Sessao de Chat

## Objetivo

Executar a user story atualmente elegivel de `OPENCLAW-MIGRATION` usando a
superficie canonica `Feature -> User Story -> Task`.

Quando a **task** alvo ja for conhecida, prefira
`PROJETOS/OPENCLAW-MIGRATION/SESSION-IMPLEMENTAR-TASK.md` (entrada `TASK_PATH`).

## Sessao Canonica

Leia e siga integralmente:

- `PROJETOS/COMUM/SESSION-IMPLEMENTAR-US.md`

## Parametros Preenchidos

```text
PROJETO:     OPENCLAW-MIGRATION
FEATURE_ID:  <resolver_na_feature_ativa_do_projeto>
US_ID:       <resolver_na_user_story_elegivel_do_projeto>
US_PATH:     <resolver_na_user_story_elegivel_do_projeto>
TASK_ID:     auto
ROUND:       1
```

## Regra Local Adicional

- descubra a unidade elegivel a partir de `PROJETOS/OPENCLAW-MIGRATION/features/`
- nao reintroduza `ISSUE-*` ou `FASE` como superficie operacional
- se a feature ainda nao tiver user stories materializadas, o proximo passo e
  completar o planejamento dessa feature antes de executar
