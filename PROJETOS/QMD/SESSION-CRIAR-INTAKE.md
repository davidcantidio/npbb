# SESSION-CRIAR-INTAKE - Criacao de Intake em Sessao de Chat

## Objetivo

Revisar ou expandir o intake do projeto QMD mantendo a fronteira entre memoria derivada e artefatos canonicos.

## Sessao Canonica

Leia e siga integralmente:

- `PROJETOS/COMUM/SESSION-CRIAR-INTAKE.md`

## Parametros Preenchidos

```text
PROJETO:       QMD
INTAKE_KIND:   new-capability
SOURCE_MODE:   original
ORIGEM:        projeto de memoria operacional com backend QMD e governanca HITL para conflitos com canonicos
ORIGIN_AUDIT:  nao_aplicavel
CONTEXT:       memoria em camadas, QMD, consolidacao noturna, conflito memoria-vs-canonico e referencias atuais em README/openclaw-workspace/README/bin/validate-host.sh
OBSERVACOES:   manter canonicos em PROJETOS/ como autoridade; qualquer conflito com memoria sobe ao PM
```

## Regra Local Adicional

O intake atual ja e valido. Use este wrapper apenas para backfill ou ampliacao de escopo, nao para reiniciar o projeto.
