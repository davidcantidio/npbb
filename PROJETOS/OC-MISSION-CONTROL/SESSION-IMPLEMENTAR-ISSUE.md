# SESSION-IMPLEMENTAR-ISSUE - Execucao de Issue em Sessao de Chat

## Objetivo

Preparar a próxima execução de issue do projeto sem regredir para o bootstrap da F1.

## Sessao Canonica

Leia e siga integralmente:

- `PROJETOS/COMUM/SESSION-IMPLEMENTAR-ISSUE.md`

## Parametros Preenchidos

```text
PROJETO:     OC-MISSION-CONTROL
FASE:        <resolver_na_fila_pos_auditoria_de_F2>
ISSUE_ID:    <resolver_na_fila_pos_auditoria_de_F2>
ISSUE_PATH:  <resolver_na_fila_pos_auditoria_de_F2>
TASK_ID:     <resolver_na_fila_pos_auditoria_de_F2>
```

## Regra Local Adicional

- o bootstrap da F1 ja foi reconciliado e aprovado; nao reutilize este wrapper para voltar a `F1-FUNDACAO`
- a fase ativa do projeto esta em `F2-OPENROUTER-E-AGENTES` e o ponto atual da fila e auditoria, nao execucao de issue
- so preencha os sentinelas acima depois da auditoria F2 ou de follow-up formal que reabra a execucao
